When making changes to an existing file, break the work into focused chunks instead of rewriting the whole file.

For each chunk:
1. State what the chunk does in one sentence before making the edit
2. Make one targeted Edit (or a small set of related Edits) that covers only that change
3. Mark the chunk done, then move to the next

Aim for roughly 5 chunks per task. Good boundaries are: signature changes, new data structures, call-site updates, logic rewrites, and comment/label cleanup. Never use Write to overwrite a file you already read — always prefer Edit.
